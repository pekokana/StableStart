「文明型・観戦競馬史シミュレーション」という壮大なコンセプトに基づき、システムの肝となる**「血統システム」と「データ永続化（SQLite/HTMX）」**への拡張について、ソースの設計思想を反映した具体的な整理を行います。

### 1. 血統システムの拡張：三代血統と個性の継承
「王朝の誕生」や「名門の崩壊」を表現するためには、単なる数値の継承ではなく、家系としての繋がりをデータ化する必要があります。

*   **データ構造の変更**: 現在の `create_horse` 関数を、父母のIDを保持する形式へ拡張します。
    *   `pedigree`: `[父, 母, 父父, 父母, 母父, 母母, ...]` のように、三代前（計14頭の祖先）までのIDリストを保持させます。
*   **遺伝的特徴（Trait）の導入**:
    *   スピード・スタミナの基礎値に加え、**「早熟」「晩成」「道悪得意」「勝負根性」**といった**良し悪しのある特徴**をフラグとして持たせます。
    *   **継承アルゴリズム**: 父から50%、母から50%の確率で引き継ぐだけでなく、三代前までの祖先が持つ特徴が「隔世遺伝」として低確率で発現する仕組みを導入します。これにより、特定の血統から突如「怪物」が生まれるドラマを演出できます。

### 2. エージェントの性格による繁殖戦略の差別化
ソース に定義された調教師（Trainer）の性格（A〜D）を、繁殖の意思決定に直結させます。

*   **A. 善意型（バランス）**: 自厩舎の牝馬の弱点を補う種牡馬を選択。
*   **B. 成長型（特化）**: スピード特化の牝馬には、さらにスピードに秀でた種牡馬を配合し、極端な個体（スプリンター）を作ろうとする。
*   **C. 多様性型（ランダム）**: アウトブリード（血の遠い配合）を優先し、予期せぬ変異を狙う。
*   **D. 自己保存型（トレンド追随）**: 現在の距離トレンドで最も賞金を稼いでいる「流行種牡馬」を優先的に選択する。

### 3. SQLiteによるデータ永続化：膨大な歴史の保存
10,000年分（あるいは1,000世代）の全頭データと血統をテキストで追うのは不可能なため、リレーショナルデータベース（SQLite）への移行は必須です。

*   **推奨されるテーブル構成**:
    *   `simulations`: シミュレートごとの設定や実行日時。
    *   `horses`: 競走馬・繁殖馬の全ステータス、戦績、血統ID。
    *   `pedigrees`: 各馬の祖先IDの紐付け。
    *   `generation_logs`: 各世代の距離トレンドや「年間ハイライト」の文章データ。
*   **メリット**: 特定の「伝説の馬」からその全子孫をSQL一発で抽出できるようになり、家系図の構築が容易になります。

### 4. HTMXを利用したブラウザ閲覧システム
「実況駆動型」のゲーム体験を実現するために、HTMXを活用した軽量な閲覧環境を構築します。

*   **「歴史を観る」インターフェース**:
    1.  **シミュレーション選択**: 過去に実行したシミュレートをキー（日時など）で選択。
    2.  **実況タイムライン**: HTMXの `hx-get` を使い、スクロールに応じて世代ごとのログ（「◯◯王朝が誕生した」等）を動的に読み込み。
    3.  **ドリルダウン**: 気になる馬名をクリックすると、その馬の三代血統表と全戦績がその場で展開されるUI。
*   **グラフの統合**: ソース で推奨されている「グラフ＋テキストのハイブリッド」を実現するため、Plotly等のライブラリと組み合わせて、世代ごとの能力分布（スピード vs スタミナ）を可視化します。

```python
def create_foundation_horse(strategy, gen):

    global NEXT_HORSE_ID

    horse_id = NEXT_HORSE_ID
    NEXT_HORSE_ID += 1

    speed = random.uniform(0.4, 0.6)
    stamina = random.uniform(0.4, 0.6)

    if strategy == "speed":
        speed += 0.1
    elif strategy == "stamina":
        stamina += 0.1

    traits = set()
    if random.random() < 0.2:
        traits.add(random.choice(TRAITS))

    return {
        "id": horse_id,
        "generation": gen,
        "speed": min(1, speed),
        "stamina": min(1, stamina),
        "traits": traits,
        "father": None,
        "mother": None,
        "pedigree": []
    }
```

## ✅ 繁殖処理改造

変更：

```python
def breed(father, mother, gen):

    global NEXT_HORSE_ID

    horse_id = NEXT_HORSE_ID
    NEXT_HORSE_ID += 1

    speed = (father["speed"] + mother["speed"]) / 2
    stamina = (father["stamina"] + mother["stamina"]) / 2

    speed += random.gauss(0, MUTATION)
    stamina += random.gauss(0, MUTATION)

    # =========================
    # Trait継承
    # =========================
    traits = set()

    # 直接継承
    for t in father["traits"]:
        if random.random() < 0.5:
            traits.add(t)

    for t in mother["traits"]:
        if random.random() < 0.5:
            traits.add(t)

    # 隔世遺伝（祖父母）
    ancestors = father["pedigree"][:4] + mother["pedigree"][:4]
    for anc_trait in ancestors:
        if random.random() < 0.05:
            traits.add(anc_trait)

    # =========================
    # pedigree生成
    # =========================
    pedigree = [father["id"], mother["id"]]

    pedigree += father["pedigree"][:6]
    pedigree += mother["pedigree"][:6]

    pedigree = pedigree[:14]

    return {
        "id": horse_id,
        "generation": gen,
        "speed": max(0, min(1, speed)),
        "stamina": max(0, min(1, stamina)),
        "traits": traits,
        "father": father["id"],
        "mother": mother["id"],
        "pedigree": pedigree
    }
```

## 子孫検索SQL（伝説血統）

```sql
WITH RECURSIVE descendants(id) AS (
    SELECT id FROM horses WHERE id = ?
    UNION
    SELECT h.id FROM horses h
    JOIN descendants d ON h.father = d.id OR h.mother = d.id
)
SELECT * FROM descendants;
```
